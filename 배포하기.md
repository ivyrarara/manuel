# 마늘 배포하기

터미널 안 씁니다. 전부 웹사이트에서 클릭으로만 합니다. 처음이면 1~2시간 잡으세요.

준비물: 이메일, 카드 (Railway 월 $5), 그리고 인내심.

---

## 1단계 — 텔레그램 봇 만들기 (5분)

1. 텔레그램에서 **@BotFather** 검색 → 대화 시작
2. `/newbot` 입력
3. 봇 이름 물어보면: `마늘`
4. 사용자명 물어보면: `아무거나_bot` (반드시 `bot`으로 끝나야 함, 이미 있는 이름이면 다른 걸로)
5. **토큰**을 줍니다. `123456:ABC-DEF...` 이렇게 생긴 거요.

> 이 토큰 = 봇의 비밀번호입니다. 카톡이나 메모장에 두지 마세요. 잠깐 어딘가 적어두고 3단계에서 쓰고 지우세요.

---

## 2단계 — Anthropic API 키 받기 (5분)

1. https://console.anthropic.com 가입
2. 결제 수단 등록 (마늘은 하루 몇 센트 수준이에요)
3. **API Keys** → **Create Key** → 복사

> 이 키도 한 번만 보여줍니다. 놓치면 새로 만들면 돼요.

---

## 3단계 — GitHub에 코드 올리기 (15분)

터미널 없이 드래그로 합니다.

1. https://github.com 가입
2. 오른쪽 위 **+** → **New repository**
3. 이름: `maneul` / **Private** 선택 (중요!) → **Create repository**
4. 다음 화면에서 **uploading an existing file** 링크 클릭
5. `maneul` 폴더를 열어서 **안에 있는 것들을 전부 선택**해서 브라우저 창에 드래그
   - `agent` 폴더, `Procfile`, `requirements.txt`, `.python-version`, `.env.example`, `.gitignore`
   - ⚠️ **`.env` 파일은 올리지 마세요.** 없을 겁니다 (안 만들었으니). 있으면 빼세요.
6. 아래 **Commit changes** 클릭

> 업로드 후 `agent` 폴더 안에 `main.py`, `db.py`, `brain.py`, `checkin.py`, `config.py`, `__init__.py` 6개가 다 있는지 확인하세요. 하나라도 없으면 안 돌아가요.

---

## 4단계 — Railway에 올리기 (20분)

1. https://railway.com → **Login with GitHub**
2. **New Project** → **Deploy from GitHub repo** → `maneul` 선택
3. GitHub 접근 권한 물어보면 허용
4. 자동으로 빌드가 시작됩니다. **여기서 한 번 실패합니다. 정상이에요.** (환경변수를 아직 안 넣었으니까요)

### 환경변수 넣기

**Variables** 탭 → **New Variable** 로 하나씩 추가:

| 이름 | 값 |
|---|---|
| `ANTHROPIC_API_KEY` | 2단계에서 받은 키 |
| `TELEGRAM_TOKEN` | 1단계에서 받은 토큰 |
| `OWNER_CHAT_ID` | `0` (일단 0, 6단계에서 고침) |
| `TIMEZONE` | `America/Toronto` |
| `TOTAL_DAYS` | `100` |
| `DB_PATH` | `/data/maneul.db` |
| `BLOG_RSS_URL` | `https://brunch.co.kr/rss/@@2Rug` |
| `GITHUB_USER` | `ivyrarara` |

> ⚠️ **`TIMEZONE`은 지금 계신 곳으로.** 이게 틀리면 밤 9시 체크인이 아침 8시에 울려요.
> 한국으로 돌아가면 이 값만 `Asia/Seoul` 로 바꾸면 됩니다.

### 볼륨 붙이기 ⚠️ 제일 중요

이거 안 하면 **재배포할 때마다 100일 기록이 전부 날아갑니다.**

1. 프로젝트 화면에서 서비스 카드 우클릭 (또는 **+ Create**) → **Volume**
2. Mount path에 정확히: `/data`
3. 저장

> `DB_PATH`가 `/data/maneul.db`이고 볼륨이 `/data`에 붙어야 합니다. 둘이 맞아야 해요.

## 5단계 — 확인

**Deployments** 탭 → 최신 배포 → **View Logs**

이게 보이면 성공:
```
마늘 시작. Day 1 / 100
```

빨간 에러가 보이면 그 텍스트를 복사해서 저한테 물어보세요.

---

## 6단계 — chat_id 넣기 (5분)

1. 텔레그램에서 1단계에서 만든 봇 찾기 (`@아무거나_bot`)
2. `/start` 보내기
3. 봇이 숫자를 알려줍니다: `당신의 chat_id: 123456789`
4. Railway **Variables** → `OWNER_CHAT_ID` 를 그 숫자로 수정
5. 자동으로 재시작됩니다

이제 아무 말이나 보내보세요. 마늘이 답하면 끝입니다.

---

## 7단계 — 블로그 · GitHub 연결 확인 (2분) ⚠️

### GitHub — `/github` 보내기

공개 저장소만 봐요. 토큰도 인증도 필요 없어요. 성공하면:

```
✅ 연결됐어요. 최근 활동 N건이 보여요.
저장소:
· ivyrarara/summer-schedule
과거 활동 N건은 성취로 세지 않았어요.
지금부터의 커밋이 5단 개선 성취로 잡혀요.
```

> **하루에 커밋을 9번 해도 성취는 1건이에요.** 커밋 수를 세면 숫자만 예뻐지니까요.
> 그리고 커밋 수는 성과가 아니라 **마찰**이에요 — 9커밋은 아홉 번 틀렸다는 뜻이니까
> `개선 (8번 만에)` 로 적혀요.

### 블로그 — `/blog` 보내기

`/blog` 를 보내세요. **이 확인을 꼭 하세요.**

RSS 주소는 추측으로 넣은 거예요 (브런치는 RSS를 공개하지 않아서, 작가 ID `@@2Rug` 로 만든 주소입니다).
틀렸으면 100일 동안 **조용히** 아무것도 안 잡혀요. 에러도 안 나요. 그게 제일 나쁜 실패예요.

**성공하면:**
```
✅ 연결됐어요. 글 2개가 보여요.
기존 글 2개는 성취로 세지 않았어요.
지금부터 올리는 글이 6단 성취로 잡혀요.
```

**실패하면**, 직접 찾아야 해요:
1. 크롬에서 `view-source:https://brunch.co.kr/@@2Rug` 열기
2. `Ctrl+F` → `rss` 검색
3. 나오는 `https://brunch.co.kr/rss/...` 주소를 복사
4. Railway `BLOG_RSS_URL` 에 붙여넣기 → `/blog` 다시

> 브런치 RSS는 **매거진이 아니라 작가 단위**예요. "클로드 플랜을 구독했다" 매거진 글만이 아니라
> 일랑님이 브런치에 쓰는 모든 글이 6단 성취로 잡혀요. 그게 싫으면 나중에 필터를 넣으면 돼요.

## 잘 되는지 보는 법

| 명령어 | |
|---|---|
| 그냥 대화 | 마늘이 응답 + 통찰/액션 |
| `/checkin` | **지금 당장** 판단시켜보기. 침묵해도 이유와 삼킨 말을 보여줌 |
| `/status` | Day N, 액션, 성취, 최근 판단 |
| `/pace` | 주차별 도달 깊이 — 페이스메이커가 보는 화면 |
| `/blog` | 블로그 RSS 확인 + 새 글 읽고 놓친 것 짚어주기 |
| `/github` | GitHub 확인 + 커밋 가져오기 |
| `/data` | 10월에 들고 갈 표의 미리보기 |
| `/actions` `/done <번호>` | 액션 관리 |
| `/prefs` `/forget <번호>` | 학습된 요구사항 관리 |

**제일 먼저 `/checkin` 눌러보세요.** 1일차엔 침묵할 겁니다 — 짚을 게 없으니까요. 그게 맞는 동작이에요.

## 판단 기준 고치기 (터미널 없이)

이게 이 프로젝트의 본체예요. 100일 동안 계속 하게 됩니다.

1. GitHub에서 `agent/checkin.py` 열기
2. 오른쪽 위 **연필 아이콘** 클릭
3. `CHECKIN_SYSTEM` 안의 판단 기준을 고침
4. 아래 **Commit changes**
5. Railway가 자동으로 재배포 → 1분 뒤 적용

> 고칠 때마다 프롬프트 버전이 자동으로 바뀝니다. `/data`에서 버전별로 몇 번 판단했는지 보여요. 이게 10월에 "튜닝해서 나아졌다"를 증명하는 근거입니다.

## 알아둘 것

- **비용**: Railway $5/월 + Claude API 하루 몇 센트.
- **DB 백업**: `/data/maneul.db` 파일 하나가 100일 전부입니다. Railway 볼륨은 안 날아가지만, 프로젝트를 실수로 지우면 끝이에요. 한 달에 한 번은 백업 방법을 찾아두세요.
- **마늘은 보이는 것만큼만 똑똑합니다.** 지금은 대화와 액션만 봅니다. 헛짚으면 대부분 프롬프트 문제가 아니라 **안 보여준 데이터** 문제예요.
- **버튼을 꼭 누르세요.** 마늘이 말을 걸 때마다 탭 한 번. 그게 안 쌓이면 10월에 코드만 남고 데이터가 없어요.
